# Definition Of Done

> ## 1. Tout le code concerné a été revu par des paires
>
> La révision du code doit être effectuée par la personne qui l'a écrit, ainsi qu'au minimum une autre personne de l'équipe.

> ## 2. Le PO (Product Owner) a vu une démonstration
>
> Dès que les attentes d'une user story sont atteintes, le product owner doit recevoir une démonstration afin de déterminer si celle-ci peut être ajoutée à la catégorie "Terminée".

> ## 3. Les standards de code sont respectés
>
> Les normes de codage de l'ETML ont été suivies.

> ## 4. Le code impliqué est stocké dans un contrôleur de source
>
> Tout le projet (code source, images, etc.) doit être stocké sur Github

> ## 5. Le produit complet est stocké dans un contrôleur de source
>
> Chaque Release doit être présent sur Github.

> ## 6. Les radiateurs d'information correspondants ont été actualisés
>
> Le Github Project et tableau physique des tâches sont régulièrement mis à jour.

> ## 7. L'information nécessaire a été communiquée au reste de l'équipe
>
> La communication à l'intérieur de l'équipe (explication fonctionnelle du livrable, considérations techniques importantes, etc.) est présente.
